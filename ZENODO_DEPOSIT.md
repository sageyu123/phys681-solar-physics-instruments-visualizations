# Zenodo Deposit Notes

This directory is prepared as a static educational resource for Zenodo archival. It is a subset
of the NJIT PHYS 681 Spring 2026 Solar Physics and Instruments teaching materials focused on
solar physics and instrumentation visualizations.

## Recommended Upload

The collection DOI is `10.5281/zenodo.20820901`; use it when citing the evolving teaching
collection. The exact v1.0.0 release is published at <https://zenodo.org/records/20820902>
with version DOI `10.5281/zenodo.20820902`. For future DOI-citable releases, prefer the
manual Zenodo upload package generated below.
It is the curated archival artifact: it excludes development pages, backup files, planning notes,
local render caches, and macOS metadata.

Use the archive generated by:

```sh
python3 tools/release/build_dist.py
```

The script writes:

- `dist/`: unpacked release candidate for local inspection.
- `zenodo-package/phys681-solar-physics-instruments-teaching-visualizations-<version>.zip`: clean archive for manual Zenodo upload.

If using the GitHub-Zenodo integration instead of manual upload, tag a release only after
confirming that the repository release contains the same curated files. A plain source archive
can include tracked development or backup files that are intentionally excluded from `dist/`.

## What The Deposit Contains

- Static HTML landing page and module indexes.
- All manifest-listed publishable visualization pages.
- Shared site CSS, navigation JavaScript, and manifest files.
- Rendered narrative notes in HTML.
- Quarto source for the narrative notes.
- PDF render of the narrative notes, linked from the HTML notes.
- Screenshot assets needed by the narrative and module cards.
- Citation, license, release, and content-manifest metadata.

## What The Deposit Excludes

- `dev/` pages.
- Backup pages.
- Planning notes.
- Local render caches.
- QA screenshots.
- macOS metadata files.

## DOI Policy

Use the Zenodo concept DOI `10.5281/zenodo.20820901` when citing the evolving collection as a
teaching resource. Use the version DOI, such as `10.5281/zenodo.20820902` for v1.0.0, when
citing an exact archived release.
