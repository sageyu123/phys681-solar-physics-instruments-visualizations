EOVSA ADC Demux-by-4

This companion explainer isolates the ingress step before the PFB/FFT: one 800 MHz ADC stream arrives faster than the ROACH FPGA fabric can process samples one-by-one, so the interface repacks the stream into 4 time samples per 200 MHz FPGA clock tick. The page shows the same 12 samples first as a serial stream, then as 4-sample words, then as the 4 parallel FPGA lanes.

Serial ADC Stream

One input stream sampled every 1.25 ns. The highlighted group is the word that will be repacked for the FPGA.

Serial interpretation: 800 MHz ADC -> one sample every 1.25 ns. This panel stays in time order only.

Demux Grouping

The same time stream is regrouped into 4-sample words so one FPGA clock edge can accept four samples at once.

Grouped interpretation: word 0 = [s0 s1 s2 s3], word 1 = [s4 s5 s6 s7], word 2 = [s8 s9 s10 s11].

FPGA Tick View

The payoff: each 200 MHz FPGA tick receives one 4-lane word from the ADC interface.

FPGA-side interpretation: 200 MHz -> one tick every 5 ns, and 4 samples/tick × 200 million ticks/s = 800 million samples/s.

Full ROACH context: one ROACH handles four input streams total (Ant 1 X, Ant 1 Y, Ant 2 X, Ant 2 Y). This page explains the demux of one stream before the PFB/FFT, not the later board-to-board corner turn.
